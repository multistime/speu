# Тікеты і бэклог

**Правілы:** `SPEU-XXX`, статусы `idea` → `todo` → `in_progress` → `review` → `done`. Ніжэй — актуальны бэклог прадукту (апісанні каротка, па кожнай задачы — змест і AC).

### SPEU-001 — Лайкі / абранае

**Змест:** для ўвайшоўшых карыстальнікаў — захаванне трэка ў «абранае» (сэрца / зорка, адзін стан на карыстальніка). Для аноніма — па кліку паказаць мадалку або банер «Увайдзіце або зарэгіструйцеся, каб захоўваць трэкі», без захавання на серверы.

**AC:** табліца/сувязь user ↔ track; ідэмпатэнтнасць паўторнага кліку (toggle); адлюстраванне стану на картцы трэка і ў плееры; анонім не атрымлівае 401 без зразумелага сцэнару.

---

### SPEU-002 — Падлік праслухоўванняў

**Змест:** падлікі па трэку. **Аўтарызаваны:** падзея «праслухоўванне» пасля **≥30 с непрарыўнага** прайгравання (адзін сеанс = адзін падлік або з дэбаўнсам); асобна **поўнае праслухоўванне** (ад пачатку да канца, з дазволам невялікай пагрэшнасці ў канцы). **Анонім:** тыя ж правілы, але захоўваем агрэгаты **па дням** (без user id), каб не раздзімаць аб’ём сырых радкоў.

**AC:** канкрэтныя парогі ў кодзе/доках; не дубляваць адзін сеанс пры паўторным seek; GDPR-friendly для анонімаў (мінімум ПД).

---

### SPEU-003 — Лічбы на песнях за 7 дзён

**Змест:** на картках трэкаў (і/або на старонках трэка/альбома) паказаць **лайкі** і **праслухоўванні** за **апошнія 7 дзён** (можна + усяго за ўвесь час дробным шрыфтам).

**AC:** кэш або матэрыялізаваны агрэгат, каб не біць БД на кожны рэндэр; зразумелая падпісь перыяду («за тыдзень»).

---

### SPEU-004 — Старонка артыста

**Змест:** асобны URL для кожнага артыста: біяграфія/апісанне, фота, спіс альбомаў і сінглаў, хіт-трэкі або апошнія рэлізы. Прадумаць візуал у стылі сайта (тыпаграфіка, сетка, мабільная версія).

**AC:** SSR/SEO meta; 404 для схаваных артыстаў; злучэнне з данымі з адмінкі.

---

### SPEU-005 — Старонка альбома

**Змест:** вокладка, назва, год, спіс трэкаў з працягласцю, прайграванне з плэйліста альбома. Трэкі **без альбома** — віртуальны альбом «Сінглы» / «Іншае» для артыста ці лэйбла (узгодніць правіла ў адмінцы).

**AC:** адзінкавы шаблон для ўсіх альбомаў; карэктныя чаргі плеера.

---

### SPEU-006 — Клікабельныя сувязі

**Змест:** усюды, дзе ў плэйлісце, спісах і картках згадваюцца артыст, альбом, трэк — зрабіць **спасылкі** на адпаведныя старонкі (уключаючы радыё/генератар, дзе ўпамінальна).

**AC:** няма «мёртвых» тэкстаў-назваў без `href` у асноўных сцэнароў.

---

### SPEU-007 — Плеер: стандартныя функцыі

**Змест:** мінімум: **queue** (чарга), **next/prev**, **паўтор / shuffle** (хоць бы базава), гучнасць, прагрэс з перамоткай, мабільны fullscreen / міні-плеер. Узгодніць з існуючым UI.

**AC:** стан захоўваецца разумна пры навігацыі па старонках (па магчымасці); клавіятурныя скароты — пажадана.

---

### SPEU-008 — Аналітыка для артыста ў кабінеце

**Змест:** для прывязанага да артыста акаўнта — графікі/лічбы: праслухоўванні, абранае, магчыма геа/крыніцы (калі ёсць даныя). Фільтр па перыядзе.

**AC:** доступ толькі да **сваіх** трэкаў (RLS/праверкі); пусты стан, калі даных няма.

---

### SPEU-009 — Агульная аналітыка лэйбла

**Змест:** у **адмінцы** — дашборд па ўсяму каталогу: топ трэкаў, дынаміка праслухоўванняў, новыя карыстальнікі/лайкі, разбіўка па артыстах.

**AC:** толькі `is_admin` (або роля лэйбла); экспарт CSV — па жаданні, не ў першым ітэрацыі.

---

### SPEU-010 — Публічная старонка лэйбла

**Змест:** адна старонка «Пра Speu / Лэйбл»: місія, артысты, апошнія рэлізы, **хіт-парад** (правіла ранжыравання задаць), спасылкі на падтрымку. Не дубляваць поўнасцю галоўную — даць унікальны карысны змест.

**AC:** уключэнне ў навігацыю/visibility па існуючай схеме `content_pages`.

---

### SPEU-011 — Падзяліцца

**Змест:** кнопкі **Share** на трэк, альбом, артыст, магчыма радыё-страніца: натыўны Web Share API дзе ёсць, інакш капіяванне спасылкі + сацыяльныя іконкі.

**AC:** карэктны `og:image` / title для шарынгу (звязаць з SEO задачамі).

---

### SPEU-012 — Галоўная: анімацыя

**Змест:** дыягнаставаць лагі/тормазы на вяліх экранах (Framer Motion, вялікія SVG, layout thrashing). Варыянты: `prefers-reduced-motion`, зніжэнне частаты анімацый, `will-change` абмежаваць, разбіць на лёгкія слоі або адключыць частку эфектаў вышэй за breakpoint.

**AC:** плавны 60 fps або прыемны статычны фолбек на 2k/ultrawide; Lighthouse performance не рэжам крытычна.

---

### SPEU-013 — Старонка трэка

**Змест:** асобны URL для трэка: вокладка альбома, артысты, прайграванне, лічбы (з SPEU-003), тэкст/спасылкі, шарынг. Калі рашэнне «не трэба» — зафіксаваць у тікеце `wontfix` з прычынай.

**AC:** canonical URL для SEO; злучэнне з плеерам.

---

### SPEU-014 — Загрузка відарысаў у кабінеце

**Змест:** для роляў, якім дазволена: **аватар артыста**, **вокладка альбома** (і магчыма банер) праз кабінет з валідацыяй памеру/фармату, захаванне ў Supabase Storage, абнаўленне запісаў у БД.

**AC:** абмежаванне памеру файла; паведамленні пра памылкі; правы доступу.

---

### SPEU-015 — Кабінет: плеер і абранае

**Змест:** у асабістым кабінеце — зручны доступ да **абранага** (спіс з плэйлістам), міні-плеер або ўбудаваны прайгравальнік па старонцы кабінета.

**AC:** сінхранізацыя з глабальным плеерам сайта (адзін крыніца праўды па «зараз гучыць»).

---

### SPEU-016 — Кабінет: перавёрстка і налады

**Змест:** прыбраць «выход» з зоны наладаў карыстальніка (вынас у хедар/меню). Асобная старонка **Налады**: рэдагаванне **імя**, магчыма email (праз Supabase), мова, абвесткі. Чыстая іерархія меню кабінета.

**AC:** валідацыя форм; пасля захавання — зразумелы фідбек; бяспека (не свеціць service keys).
