### SPEU-017 — Мета-пазнакі трэка: 18+, AI, інструментал

**Навошта:** праўда пра змест для слухача, фільтрацыя каталога, праўныя/этічныя моманты (узрост, прозорасць AI), карэктны UX для твораў без вакалу.

#### 1. Даныя (`speu.artist_tracks` або асобная табліца)

Рэкамендацыя — тры **boolean** з зразумелым сэнсам і дэфолтам `false`:

| Поле (прыклад) | Сэнс |
|----------------|------|
| `mature_content` / `is_explicit_18` | Трэк змяшчае матэрыял **18+** (моцная лексіка, тэматыка). Не змешваць з «радыё-версіяй» — гэта асобная ідэя (SPEU-018). |
| `ai_generated` | Творчасць **з удзелам AI** (тэкст, аранжыроўка, вакал-сінтэз і г.д.). Пры неабходнасці пазней: опцыянальнае кароткае поле `ai_disclosure_note` для артыста. |
| `is_instrumental` | **Без словаў** (чысты інструментал); карысна для плэйлістаў і каб схаваць/прыбраць блок «тэкст». |

**Міграцыя:** `add column … default false`; бэкфіл не абавязковы. **RLS:** публічны `select` ужо на трэках — новыя слупкі аўтаматычна ў публічным прадстаўленні, калі не выносіць у асобны view.

#### 2. Адмінка

- Форма рэдагавання трэка: тры чэкбоксы + кароткія падказкі (што лічыцца 18+, што пад AI).
- Апцыянальна: масавае рэдагаванне або CSV — у другую чаргу.

#### 3. Публічны API і тыпы

- Уключыць палё ў адказ `public` hub / `SpeuPublicTrack` (або эквівалент).
- Кэш/рэвалідацыя старонак пасля змены палёў.

#### 4. Публічны UI

- **Бейджы** на радку трэка, старонцы трэка, міні-плэеры: «18+», «AI», «інструментал» (іконкі + `title` для доступнасці).
- **18+:** перад першым прайграваннем у сеансе — мяккае пацвярджэнне або банер «змест для дарослых» (наладжвальна); для радыё — правіла фільтрацыі плэйліста.
- **AI:** спасылка на старонку палітыкі лэйбла або tooltip «частка матэрыялу створана з дапамогай AI».
- **Інструментал:** не паказваць кнопку «тэкст песні» / блок лірыкі; у мета — `og:description` без ілюзіі наяўнасці слоў.

#### 5. AC (мінімум)

- Усе тры прызнакі рэдагуюцца ў адмінцы і адлюстроўваюцца на `/speu` і старонцы трэка.
- Дэфолты бяспечныя (`false`).
- Лакалізацыя падпісаў (BE/RU) узгоднена з камандай.

#### Пашырэнні (ідэі на потым — гл. SPEU-018)

- **Мова тэксту** (`lyrics_language`: `be` / `ru` / `en` / `mixed`) — фільтры і SEO.
- **Live vs studio**, **сэмплы / інтэрпаляцыя** — для правоў і крэдытаў.
- **Radio edit / clean version** — асобны трэк або сцяг «без матаў».
- **Змест з удзелам дзяцей** — асобная пазнака, калі калі-небудзь спатрэбіцца.
